# Documentation PGYS

Ce dossier contient les documents fondateurs du projet PGYS.

## Documents

- `00-VISION.md` : vision générale de PGYS.
- `01-PRODUIT.md` : définition des produits, offres, cibles et MVP.
- `02-ARCHITECTURE.md` : architecture logicielle et technique.
- `03-BRANDING.md` : identité visuelle et ton de marque.
- `04-INFRASTRUCTURE.md` : infrastructure serveur, Docker, sauvegardes et supervision.
- `05-DEVELOPPEMENT.md` : conventions de développement.
- `06-ROADMAP.md` : feuille de route produit.
- `07-BACKLOG.md` : backlog initial.
- `08-DEPLOIEMENT.md` : stratégie de déploiement.
- `09-SECURITE.md` : règles de sécurité.

## Principe

PGYS est construit comme une plateforme de services numériques pour petites entreprises, et non comme un simple service cloud.

## Utilisation

La documentation du dépôt est la source de vérité pour les décisions produit et
techniques. Toute modification structurante doit mettre à jour le document
concerné dans la même pull request.

Pour contribuer, commencer par la vision, le produit, l'architecture et les
conventions de développement. Les règles générales de contribution sont
décrites dans [`CONTRIBUTING.md`](../CONTRIBUTING.md).
