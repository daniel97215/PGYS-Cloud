# PGYS — Roadmap

## Sprint 0 — Fondations

Objectif : poser les bases du projet.

Livrables :

- documentation ;
- structure monorepo ;
- règles IA ;
- conventions Git ;
- infrastructure existante stabilisée.

## V0 — Présence publique

Objectif : mettre en ligne un site professionnel.

Livrables :

- page d’accueil ;
- page PGYS Cloud ;
- page applications métier ;
- page hébergement ;
- page contact ;
- formulaire de demande.

## V1 — PGYS Cloud manuel

Objectif : vendre et gérer les premiers clients manuellement.

Livrables :

- Nextcloud fonctionnel ;
- offres simples ;
- création manuelle des comptes ;
- quotas ;
- sauvegardes ;
- monitoring.

## V2 — Administration interne

Objectif : réduire les actions manuelles.

Livrables :

- admin PGYS ;
- liste clients ;
- fiches clients ;
- services associés ;
- actions Nextcloud semi-automatisées.

## V3 — API PGYS

Objectif : centraliser la logique métier.

Livrables :

- NestJS ;
- PostgreSQL ;
- Prisma ;
- clients ;
- offres ;
- utilisateurs ;
- services.

## V4 — Automatisation

Objectif : automatiser le parcours client.

Livrables :

- création automatique de compte ;
- email de bienvenue ;
- gestion quota ;
- suspension ;
- renouvellement.

## V5 — Paiement

Objectif : intégrer la facturation.

Livrables :

- Stripe ;
- abonnements ;
- factures ;
- webhooks ;
- statut client.

## V6 — PGYS Apps

Objectif : proposer un portail d’applications métier.

Livrables :

- catalogue d’applications ;
- hébergement client ;
- accès centralisé ;
- support.

## V7 — PGYS Backup

Objectif : proposer la sauvegarde externalisée.

Livrables :

- agent ou procédure de sauvegarde ;
- stockage secondaire ;
- restauration ;
- supervision.
