# PGYS Services — Modèle économique initial

**Version :** 1.0
**Statut :** Brouillon stratégique à valider
**Horizon :** 12 à 24 mois
**Nature du document :** Modèle de revenus, structure d’offre et règles de rentabilité

---

# 1. Objet du document

Ce document définit la manière dont PGYS Services doit générer des revenus durant sa première phase commerciale.

Il précise :

* ce qui est inclus dans l’abonnement ;
* ce qui doit être facturé séparément ;
* comment structurer les niveaux d’offre ;
* comment éviter que l’accompagnement ne détruise la marge ;
* quels indicateurs suivre ;
* à quelles conditions un client est économiquement viable.

L’objectif est de construire un modèle qui puisse être rentable avec peu de clients, puis scalable avec plusieurs dizaines ou centaines d’entreprises.

---

# 2. Principe économique central

PGYS Services ne doit pas être vendu uniquement comme un accès à un logiciel.

L’offre doit combiner trois sources de revenus :

```text
Abonnement récurrent
        +
Mise en route facturée
        +
Services complémentaires cadrés
```

L’abonnement finance :

* l’accès au logiciel ;
* l’hébergement ;
* les mises à jour ;
* la maintenance ;
* le support standard ;
* l’amélioration continue.

La mise en route finance :

* le diagnostic ;
* le paramétrage ;
* la migration ;
* la formation ;
* le lancement du client.

Les services complémentaires financent les besoins qui dépassent le fonctionnement standard du produit.

---

# 3. Ce que PGYS vend réellement

Le client ne paie pas uniquement pour :

* des écrans ;
* des utilisateurs ;
* des factures ;
* un planning ;
* un module d’encaissement.

Il paie pour obtenir :

* une activité mieux structurée ;
* moins de ressaisies ;
* moins d’oublis ;
* une facturation plus rapide ;
* une meilleure visibilité ;
* un historique client centralisé ;
* une diminution de sa dépendance aux cahiers et fichiers ;
* un accompagnement au démarrage.

La tarification ne doit donc pas être comparée uniquement au prix d’un logiciel de facturation basique.

---

# 4. Modèle recommandé

Le modèle initial recommandé repose sur :

* un abonnement par entreprise ;
* un nombre d’utilisateurs inclus ;
* des options fonctionnelles ;
* une mise en route obligatoire ;
* des prestations facturées au-delà du périmètre standard.

Le prix ne doit pas dépendre uniquement du nombre d’utilisateurs.

La valeur de PGYS provient surtout du périmètre métier utilisé et du niveau d’accompagnement.

---

# 5. Architecture des offres

Trois niveaux sont recommandés.

---

## 5.1. Offre Essentiel

### Cible

Très petite entreprise ayant besoin de centraliser son activité principale sans organisation complexe.

### Inclus

* 1 établissement ;
* jusqu’à 2 utilisateurs ;
* clients ;
* catalogue ;
* commandes ou prestations ;
* facturation ;
* avoirs ;
* règlements multiples ;
* historique ;
* synthèse quotidienne ;
* sauvegarde ;
* support standard.

### Exclusions

* planning multi-collaborateurs ;
* stock avancé ;
* portail client ;
* automatisations ;
* accompagnement périodique ;
* paramétrage complexe.

### Hypothèse tarifaire

> **49 à 69 € HT par mois**

Le tarif plancher ne doit pas être inférieur au coût réel d’hébergement, de support et de maintenance.

---

## 5.2. Offre Professionnel

### Cible

Entreprise de services avec plusieurs utilisateurs, un planning ou un besoin opérationnel plus structuré.

### Inclus

Tout le niveau Essentiel, plus :

* jusqu’à 5 utilisateurs ;
* planning ;
* rendez-vous ;
* collaborateurs ;
* statuts personnalisables dans un cadre défini ;
* tableau de bord enrichi ;
* export comptable ;
* options de saisie tactile ;
* support prioritaire ;
* paramétrage plus avancé.

### Hypothèse tarifaire

> **89 à 129 € HT par mois**

Cette offre doit devenir le niveau principal recommandé commercialement.

---

## 5.3. Offre Partenaire

### Cible

Entreprise souhaitant être accompagnée dans l’amélioration continue de son organisation.

### Inclus

Tout le niveau Professionnel, plus :

* jusqu’à 10 utilisateurs ;
* revue régulière de l’usage ;
* accompagnement périodique ;
* aide au paramétrage ;
* formation complémentaire ;
* analyse des indicateurs ;
* coordination de certains partenaires ;
* accès prioritaire au support ;
* accès anticipé à certaines évolutions.

### Hypothèse tarifaire

> **179 à 299 € HT par mois**

Cette offre ne doit pas devenir une assistance illimitée.

Le nombre d’heures ou d’interventions incluses doit être explicitement plafonné.

---

# 6. Utilisateurs supplémentaires

Les utilisateurs supplémentaires peuvent être facturés lorsque le nombre inclus est dépassé.

Hypothèse :

> **8 à 15 € HT par utilisateur et par mois**

Cependant, PGYS ne doit pas pénaliser excessivement l’adoption interne.

Il peut être pertinent de distinguer :

* utilisateur complet ;
* utilisateur opérationnel ;
* utilisateur en lecture seule ;
* accès ponctuel.

Les accès en lecture seule peuvent être gratuits ou peu coûteux afin de favoriser la diffusion du produit.

---

# 7. Options fonctionnelles

Les options doivent compléter l’offre sans créer une tarification illisible.

## Options possibles

* planning avancé ;
* stock ;
* achats ;
* intervention mobile ;
* portail client ;
* réservation en ligne ;
* fidélité ;
* cartes cadeaux ;
* automatisations ;
* SMS ;
* stockage supplémentaire ;
* plusieurs établissements.

## Hypothèse tarifaire

Chaque option peut être facturée :

> **10 à 40 € HT par mois**

Les options à forte valeur ou à coût variable peuvent être facturées davantage.

PGYS doit limiter le nombre d’options visibles.

Une offre composée de quinze micro-options serait difficile à vendre et à comprendre.

---

# 8. Mise en route obligatoire

La mise en route doit être facturée séparément.

Elle est indispensable pour :

* garantir la qualité du déploiement ;
* financer le temps initial ;
* responsabiliser le client ;
* éviter que l’abonnement absorbe plusieurs jours de travail ;
* standardiser l’onboarding.

---

## 8.1. Mise en route Essentiel

### Inclus

* diagnostic court ;
* création de l’espace ;
* paramétrage standard ;
* import simple des clients et articles ;
* configuration des documents ;
* formation initiale ;
* validation du cycle principal.

### Hypothèse tarifaire

> **390 à 690 € HT**

---

## 8.2. Mise en route Professionnel

### Inclus

* diagnostic détaillé ;
* paramétrage ;
* reprise de données standard ;
* configuration du planning ;
* modèles de documents ;
* formation par profil ;
* assistance au démarrage ;
* vérification après lancement.

### Hypothèse tarifaire

> **790 à 1 490 € HT**

---

## 8.3. Mise en route Partenaire

### Inclus

* analyse des processus ;
* plan de déploiement ;
* migration plus large ;
* paramétrage avancé ;
* formation ;
* accompagnement renforcé ;
* revue après mise en production.

### Hypothèse tarifaire

> **1 500 à 3 000 € HT**

Cette offre doit rester cadrée.

Toute intégration ou migration exceptionnelle doit faire l’objet d’un devis distinct.

---

# 9. Migration des données

La migration est l’un des principaux risques économiques.

Elle doit être normalisée.

## Migration standard

Peut inclure :

* clients ;
* fournisseurs ;
* articles ;
* prestations ;
* soldes ou documents ouverts lorsque le format le permet.

Elle doit utiliser :

* un modèle CSV ;
* un format défini ;
* une procédure répétable ;
* une validation par le client.

## Migration complexe

Doit être facturée séparément lorsqu’elle implique :

* extraction depuis un ancien logiciel ;
* nettoyage important ;
* transformation de données ;
* reprise historique ;
* dédoublonnage ;
* rapprochements ;
* corrections manuelles.

La migration complexe peut être facturée :

* au forfait ;
* à la journée ;
* selon le volume ;
* après audit.

Elle ne doit jamais être incluse implicitement dans un abonnement.

---

# 10. Formation

La formation initiale doit être incluse dans la mise en route.

Les formations complémentaires doivent être facturées séparément.

## Exemples

* formation d’un nouveau salarié ;
* formation avancée ;
* formation sur site ;
* atelier d’amélioration des processus ;
* création de supports spécifiques.

Hypothèse tarifaire :

> **90 à 150 € HT par heure**

ou :

> **450 à 900 € HT par journée**

Les tarifs exacts dépendront du mode de réalisation et du déplacement éventuel.

---

# 11. Support

Le support doit être clairement distingué de la formation, du conseil et du paramétrage.

## Support standard

Couvre :

* incidents ;
* bugs ;
* questions sur les fonctions existantes ;
* aide ponctuelle ;
* accès à la documentation.

## Ne relève pas du support standard

* création complète de nouveaux paramétrages ;
* import massif ;
* modification des processus ;
* formation répétée ;
* analyse métier ;
* intervention chez le client ;
* développement spécifique.

Cette distinction doit être expliquée dès la vente.

---

# 12. Services complémentaires

PGYS peut facturer séparément :

* migration complexe ;
* intégration avec un outil externe ;
* modèles documentaires spécifiques ;
* formation complémentaire ;
* audit de processus ;
* assistance sur site ;
* reprise ou nettoyage de données ;
* création de rapports ;
* accompagnement renforcé ;
* coordination de partenaires.

Ces services doivent rester compatibles avec la stratégie produit.

PGYS ne doit pas accepter une prestation qui l’oblige à maintenir une version différente du logiciel.

---

# 13. Remises et promotions

Les remises doivent rester exceptionnelles et encadrées.

## Remises acceptables

* engagement annuel ;
* client pilote ;
* offre de lancement limitée ;
* recommandation ou partenariat ;
* paiement anticipé.

## Remises à éviter

* remise permanente sans contrepartie ;
* négociation systématique ;
* gratuité prolongée ;
* personnalisation offerte ;
* baisse de prix pour compenser un produit non adapté.

Une remise doit toujours avoir :

* une durée ;
* une justification ;
* une contrepartie ;
* une date de fin.

---

# 14. Engagement annuel

PGYS peut proposer :

* paiement mensuel sans engagement ;
* engagement annuel avec remise ;
* paiement annuel anticipé.

Hypothèse :

> **10 à 15 % de remise pour un paiement annuel**

L’engagement annuel améliore :

* la trésorerie ;
* la prévisibilité ;
* la rétention ;
* l’amortissement de l’onboarding.

Mais il ne doit pas servir à enfermer un client insatisfait.

---

# 15. Essai gratuit

Un essai gratuit autonome n’est pas recommandé dans la première phase.

Le produit nécessite :

* du paramétrage ;
* de la compréhension métier ;
* parfois une migration ;
* une formation.

Un essai non accompagné pourrait donner une mauvaise image du produit.

## Alternative recommandée

* démonstration personnalisée ;
* atelier de découverte ;
* environnement de démonstration ;
* période pilote payante ou faiblement facturée ;
* garantie de satisfaction limitée.

Une offre pilote peut être proposée avec un périmètre et une durée précis.

---

# 16. Offre pilote

L’offre pilote peut être utilisée pour les premiers clients externes au cercle initial.

## Caractéristiques

* durée de 2 à 3 mois ;
* périmètre limité ;
* objectifs définis ;
* prix réduit mais non gratuit ;
* échanges réguliers ;
* droit d’utiliser le retour client comme étude de cas, avec accord ;
* passage à l’offre normale si les objectifs sont atteints.

Hypothèse :

> **50 % du tarif de mise en route et abonnement réduit pendant la période pilote**

Cette remise rémunère la contribution du client sans dévaloriser le produit.

---

# 17. Règles de rentabilité par client

Un client ne doit pas être évalué uniquement sur son chiffre d’affaires mensuel.

Il faut mesurer :

* revenu d’abonnement ;
* revenu de mise en route ;
* temps de support ;
* temps de formation ;
* coût d’hébergement ;
* coût des services tiers ;
* temps de développement spécifique ;
* risque de churn ;
* potentiel de recommandation ;
* valeur stratégique du segment.

## Formule simplifiée

```text
Marge client =
revenus encaissés
- coûts techniques
- temps de support
- temps d’accompagnement
- coûts spécifiques
```

Un client qui paie 100 € par mois mais consomme 6 heures de support n’est pas rentable.

---

# 18. Seuil minimal de rentabilité

Dans la phase initiale, PGYS doit viser une marge brute élevée sur l’abonnement.

Objectif recommandé :

> **Au moins 75 % de marge brute sur le revenu récurrent hors temps exceptionnel de développement.**

La mise en route doit couvrir au minimum :

* le temps de diagnostic ;
* le paramétrage ;
* la migration standard ;
* la formation ;
* le suivi initial.

Aucun onboarding ne doit être vendu à perte dans l’espoir d’une rentabilité future incertaine.

---

# 19. Coût maximal d’accompagnement

Le temps de support et d’accompagnement inclus doit être plafonné.

Hypothèse de travail :

## Offre Essentiel

* moins de 30 minutes de support moyen par mois.

## Offre Professionnel

* moins d’une heure de support moyen par mois.

## Offre Partenaire

* un volume explicite, par exemple 2 à 3 heures par mois.

Au-delà, le temps supplémentaire doit être :

* facturé ;
* traité par une formation ;
* transformé en amélioration produit ;
* ou faire l’objet d’un recadrage.

---

# 20. Valeur vie client

PGYS doit chercher une relation de plusieurs années.

Exemple théorique :

```text
Abonnement moyen : 100 € / mois
Durée moyenne : 5 ans
Revenu récurrent cumulé : 6 000 €
Mise en route : 1 000 €
Services complémentaires : 1 000 €

Valeur totale du client : 8 000 €
```

Cette logique montre pourquoi :

* la rétention ;
* la qualité ;
* l’accompagnement initial ;
* l’évolution du produit ;

sont plus importants qu’une vente ponctuelle.

---

# 21. Objectif de revenu récurrent

La priorité n’est pas seulement le chiffre d’affaires total.

PGYS doit suivre le revenu mensuel récurrent.

## Paliers indicatifs

### Palier 1 — Validation

* 5 clients payants ;
* panier moyen supérieur à 80 € ;
* onboarding reproductible ;
* support maîtrisé.

### Palier 2 — Premiers revenus stables

* 20 clients ;
* 2 000 à 3 000 € de revenu mensuel récurrent ;
* premiers témoignages ;
* offre stabilisée.

### Palier 3 — Validation commerciale

* 50 clients ;
* 5 000 à 7 500 € de revenu mensuel récurrent ;
* acquisition moins dépendante du réseau personnel ;
* faible churn.

### Palier 4 — Première base scalable

* 100 clients ;
* 10 000 à 15 000 € de revenu mensuel récurrent ;
* onboarding documenté ;
* support structuré ;
* offre verticale validée.

Ces seuils sont indicatifs et devront être ajustés selon les prix réellement acceptés.

---

# 22. Indicateurs économiques à suivre

PGYS doit suivre au minimum :

* revenu mensuel récurrent ;
* revenu annuel récurrent ;
* panier moyen par client ;
* revenu de mise en route ;
* marge brute ;
* coût d’hébergement par client ;
* temps de support par client ;
* coût d’acquisition ;
* taux de conversion ;
* durée du cycle de vente ;
* churn ;
* taux de renouvellement ;
* nombre de modules par client ;
* revenu des options ;
* délai de récupération du coût d’acquisition.

---

# 23. Conditions d’augmentation des prix

Les prix pourront augmenter lorsque :

* la valeur mesurable augmente ;
* de nouveaux modules importants sont inclus ;
* l’onboarding devient plus performant ;
* les clients acceptent rapidement l’offre ;
* le support reste maîtrisé ;
* le positionnement devient plus spécialisé ;
* la demande dépasse la capacité disponible.

Les clients historiques peuvent bénéficier d’une période de transition.

Mais PGYS ne doit pas promettre de maintenir éternellement des tarifs trop faibles.

---

# 24. Risques du modèle économique

## 24.1. Prix trop bas

Un prix trop faible attire des clients sensibles au coût et laisse peu de marge pour l’accompagnement.

## 24.2. Services illimités

Un accompagnement non cadré transforme l’abonnement en prestation déficitaire.

## 24.3. Trop d’options

Une grille trop complexe ralentit la vente et rend la facture incompréhensible.

## 24.4. Mise en route gratuite

Une mise en route offerte dévalorise le travail initial et crée un risque de perte.

## 24.5. Clients pilotes éternellement gratuits

Un pilote doit finir par payer.

Sinon, il ne valide pas réellement la valeur du produit.

## 24.6. Personnalisation cachée

Des adaptations non facturées peuvent consommer la totalité de la marge.

---

# 25. Règles commerciales

* Toute offre comporte un abonnement.
* Toute mise en route significative est facturée.
* Toute migration complexe fait l’objet d’un devis.
* Toute demande spécifique est séparée de l’abonnement.
* Les remises sont limitées dans le temps.
* Le support n’est pas du conseil illimité.
* Les options sont peu nombreuses et lisibles.
* Le niveau Professionnel est l’offre principale.
* L’offre Partenaire doit être fortement valorisée.
* Aucun client ne bénéficie indéfiniment d’un tarif pilote.

# 27. Test de sensibilité

Avant validation, PGYS devra tester plusieurs hypothèses.

## Hypothèse basse

* abonnement moyen : 70 € ;
* forte sensibilité au prix ;
* accompagnement limité.

## Hypothèse centrale

* abonnement moyen : 110 € ;
* mise en route autour de 1 000 € ;
* offre Professionnel majoritaire.

## Hypothèse haute

* abonnement moyen : 180 € ;
* accompagnement fort ;
* cible plus exigeante ;
* promesse de valeur plus importante.

La stratégie recommandée est de tester d’abord l’hypothèse centrale.

---

# 28. Validation du pricing

Le modèle sera considéré comme validé lorsque :

1. plusieurs clients acceptent le même niveau de prix ;
2. le prix n’exige pas de justification excessive ;
3. la mise en route couvre réellement le temps consommé ;
4. le support reste compatible avec la marge ;
5. les clients perçoivent une valeur supérieure au coût ;
6. la majorité choisit une offre standard ;
7. les remises restent minoritaires ;
8. le renouvellement est naturel ;
9. le produit réduit réellement des coûts ou du temps ;
10. le modèle peut être reproduit sur plusieurs clients.

---

# 29. Décisions actées

* PGYS Services reposera sur un abonnement récurrent.
* La mise en route sera obligatoire et facturée.
* Trois niveaux d’offre seront utilisés : Essentiel, Professionnel et Partenaire.
* L’offre Professionnel sera l’offre centrale.
* Les options resteront limitées.
* La migration complexe sera facturée séparément.
* Le support sera distingué de la formation et du conseil.
* Les clients pilotes devront progressivement rejoindre un modèle payant.
* Les remises seront temporaires et conditionnelles.
* La rentabilité sera suivie client par client.
* Le panier moyen cible initial devra être proche ou supérieur à 100 € HT par mois.
* La marge ne devra pas être sacrifiée pour accélérer artificiellement l’acquisition.

---

# 30. Questions à trancher

* Les clients pilotes paient-ils déjà le logiciel ou les prestations associées ?
* Quel prix payaient-ils auparavant pour EBP et les autres outils ?
* Combien de temps PGYS consacre-t-il actuellement au support de chaque client ?
* Quel coût réel représente la mise en route ?
* Le planning doit-il être inclus dans Professionnel ou facturé en option ?
* Quel niveau d’accompagnement le salon de coiffure attend-il ?
* Le support téléphonique sera-t-il inclus ?
* Une assistance sur site sera-t-elle proposée ?
* Les tarifs doivent-ils varier selon le nombre d’établissements ?
* Quel prix minimal le marché accepte-t-il sans négociation excessive ?

---

# 31. Formulation économique synthétique

> **PGYS Services associe un abonnement logiciel récurrent à une mise en route facturée et à des services complémentaires cadrés. Le modèle privilégie la valeur, la rétention et la rentabilité plutôt que les prix bas et les prestations illimitées.**

---

# 32. Principe directeur

> **Un client n’est pas rentable parce qu’il paie un abonnement. Il est rentable lorsque la valeur qu’il perçoit reste durablement supérieure au coût qu’il génère pour PGYS.**
